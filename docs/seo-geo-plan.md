# 于承惠双手剑 SEO / GEO 实施方案

更新日期：2026-07-20

状态：前期设计基线

## 1. 范围与原则

本文中的 GEO 指生成式引擎优化（Generative Engine Optimization），不是地理位置优化。

SEO 与 GEO 共用同一基础：让真实、原创、有出处的内容可以被用户、搜索引擎和生成式系统稳定读取与引用。Google 对 AI 搜索功能的官方说明明确表示，AI 概览和 AI 模式没有额外的技术要求，仍应遵循基础 SEO 最佳实践。

本站不采用：

- 关键词堆砌或批量生成近义页面。
- 隐藏文本、隐藏问答或与可见正文不一致的结构化数据。
- 未经校订却写成定论的动作名称、歌诀或历史关系。
- 从其他网站大量改写、拼接而缺少本站原创价值的内容。
- 为统计或排名保存用户学习进度、建立用户画像。
- 宣称能够保证排名或被生成式答案引用。

## 4. 页面内容模板

### 4.1 所有内容页

页面首段直接回答“这是什么”，控制在约 40-100 个汉字，不使用空泛宣传语。

随后依次提供：

1. 核心定义或结论。
2. 可见的视频、图片或古籍图版。
3. 分步骤讲解或原文。
4. 术语解释和相关页面链接。
5. 来源、版本、编辑状态和更新时间。
6. 仍有争议或待校订的内容。

页面必须使用唯一 `h1`、顺序清楚的章节标题、可抓取的普通 HTML 文本和描述性链接。重要信息不能只存在于视频、图片、Canvas 或客户端弹窗中。

### 4.2 动作教学页

固定回答以下问题：

- 动作名称是什么，当前是否已经校订？
- 它属于哪一段、哪一组、序号是多少？
- 使用了击、刺、格、洗中的哪类基本技法？
- 视频从第几秒开始，到第几秒结束？
- 起势、过渡和定势分别是什么？
- 剑尖轨迹、步法和身体转向如何？
- 初学者容易出现什么问题？
- 说明依据来自原始影像、出版资料还是编辑示意？

### 4.3 古谱单势页

固定包含：

- 势名及所属技法。
- 图版、原文和现代标点。
- 白话解释与术语注释。
- 图版底本、卷次和页码；未确认时明确写“待核验”。
- 现代考释与原文分栏或分节，不混为原典原意。
- 与其他势或现代套路的相关点；没有证据时不建立映射。

### 4.4 人物与史料页

- 使用明确日期、机构、书名和出版信息。
- 每个关键生平节点附来源。
- 区分“原始资料记载”“后世称谓”“本站编辑概述”。
- 人物照片、视频截图和书封注明出处及许可。

## 5. GEO 内容规则

生成式系统更容易引用边界清楚、来源透明、可独立理解的内容块。本站采用以下规则：

- 每个章节只解决一个明确问题，并设置稳定锚点。
- 关键事实使用完整主语，例如“视频字幕称该套路共四段、40 个动作”，避免脱离上下文的“共 40 个”。
- 数字、名称、年代旁边提供来源链接或脚注。
- 对存在争议的结论同时说明已知事实、不同观点和本站处理方式。
- 原创价值优先来自视频精确时间码、动作拆解、人工校订、教学插图和古谱版本对照。
- 提供可复制的文字稿，不把核心内容锁在 PDF 或图片里。
- 术语首次出现时给出简短定义，并链接到术语中心页。
- 页面显示作者/整理者、校订状态、首次发布日期和最近实质更新日期。
- 结构化数据、标题、摘要与可见正文使用一致的实体名称。
- 问答内容必须真实出现在页面中，不为结构化数据虚构 FAQ。

### 5.1 可直接回答的核心问题

这些问题应由中心页正文自然回答，用于覆盖初学者搜索和生成式搜索的查询扇出：

- 于承惠双手剑是什么？
- 于承惠双手剑共有多少段、多少动作？
- 双手剑的击、刺、格、洗分别是什么？
- 初学双手剑需要什么器械和场地？
- 于承惠双手剑如何分段学习？
- 《武备志》中的朝鲜势法是什么？
- 朝鲜势法二十四势有哪些？
- 朝鲜势法与于承惠双手剑是不是同一套套路？
- 视频最后两篇歌诀是什么，哪些文字仍待校订？

答案必须从本站已核验资料出发，不为覆盖关键词而扩大结论。

## 6. 视频 SEO

视频是本站的主要原创组织价值，需要独立设计搜索入口。

### 6.1 观看页要求

- 每个主要视频或动作组拥有可直接观看的独立页面。
- 视频位于页面主要内容区域，不藏在折叠面板或点击后才创建的 DOM 中。
- 页面附近提供描述性标题、摘要、字幕和动作说明。
- 使用稳定的 `contentUrl`、`embedUrl`、缩略图 URL、上传/发布日期和时长。
- 为视频生成唯一、清晰且可抓取的缩略图。
- 输出 WebVTT 字幕和普通 HTML 文字稿。
- 使用视频站点地图，列出重点观看页。

### 6.2 `VideoObject + Clip`

主教学视频的 `VideoObject` 内嵌 `Clip`，标记：

- 历史与创传导读。
- 第一遍整套演练。
- 击、刺、格、洗四法。
- 四段各动作组。
- 整套演示。
- 单练行功歌诀。
- 特点与演练要诀歌（剑论）。

每个 `Clip` 包含唯一名称、开始秒数、结束秒数和带时间参数的页面 URL。动作名称处于“待校订”状态时，结构化数据使用页面可见的同一名称和状态说明，校订后再同步更新。

如页面支持 `?t=秒数` 或 `#t=秒数`，可同时评估 `SeekToAction`。首版优先使用人工确定的 `Clip`，因为动作时间码已经由本站整理。

## 7. 图片 SEO 与古谱图版

- 图片文件名使用序号和稳定 slug，例如 `01-juding.webp`。
- `alt` 描述图中实际姿态，例如“朝鲜势法举鼎势古谱图，人物双手举剑过头”。
- 图注显示势名、图版来源、版本和“原图 / 编辑示意”状态。
- 不在 `alt` 中堆叠“于承惠、双手剑、教学、武术”等无关关键词。
- 使用 `width`、`height`、`srcset` 和 `sizes`，避免布局跳动。
- 生产版本优先使用 AVIF/WebP，保留原始扫描作为研究母版。
- 古谱页面可使用 `ImageObject`，但许可、来源、宽高和内容必须与页面一致。
- 知乎归档图当前仅作研究参考；找到已核权公版底本前，不作为正式搜索图片资产发布。

## 8. 技术 SEO 基线

### 8.1 抓取与索引

- 使用静态生成或服务端渲染，核心正文在初始 HTML 中存在。
- 提供 `robots.txt`，并在其中声明 sitemap。
- 输出 sitemap 索引、内容 sitemap 和视频 sitemap。
- 每个正式页面设置唯一 canonical URL。
- 草稿、预览和重复筛选页使用 `noindex`，不能只依靠 `robots.txt` 阻止索引。
- 旧 URL 使用 301 重定向到唯一新地址。
- 站内所有重点页面至少有一个可抓取的普通链接入口。
- `lastmod` 只在内容发生实质变化时更新。

### 8.2 元数据

每页必须提供：

- 唯一 `<title>`，自然包含页面主体名称。
- 准确的 meta description，不使用相同模板批量复制。
- canonical、`lang="zh-CN"`、Open Graph 和分享图。
- 可见面包屑与 `BreadcrumbList`。
- 适用页面的 JSON-LD。

不使用 `meta keywords`。标题中不堆砌同义关键词，也不为了排名在域名中强塞关键词。

### 8.3 性能与体验

- 目标 Core Web Vitals：LCP <= 2.5s、INP <= 200ms、CLS <= 0.1（以 75 分位真实用户数据评估）。
- 首屏主图使用响应式压缩格式并预留尺寸。
- 非首屏古谱图和动作图延迟加载。
- 视频不自动播放带声音，不阻塞正文首屏渲染。
- 字体使用 `font-display: swap`，只预加载必要字重。
- 375px 手机和横屏均不得出现横向滚动或固定控件遮挡。

## 9. AI 抓取与发布政策

- Google 的 AI 搜索功能遵循 Googlebot 的正常抓取和预览控制。
- 如需限制 Google 搜索摘要，可使用 `nosnippet`、`data-nosnippet`、`max-snippet` 或 `noindex`，但这会影响传统搜索和 AI 搜索展示。
- `Google-Extended` 控制部分非搜索用途，不等同于禁止 Google 搜索索引；上线前单独决定政策。
- 其他生成式平台的抓取策略在上线前建立清单，分别决定允许搜索引用、允许训练或全部拒绝。
- 可提供实验性的 `/llms.txt`，列出站点名称、核心实体、权威中心页和引用规则，但它不是通用排名标准，也不能替代 sitemap、robots 或正常内部链接。
- 公开授权、引用方式和内容许可应使用人可读页面表达，必要时再补机器可读许可字段。

## 10. 发布与监测

本站不保存学习进度，也不需要用户级行为画像。SEO 监测使用聚合数据：

- Google Search Console：索引、搜索词、页面、视频和 Core Web Vitals。
- Bing Webmaster Tools：抓取、索引和搜索表现。
- 百度搜索资源平台：站点验证、链接提交和抓取异常。
- 服务器/CDN 聚合日志：状态码、爬虫访问和资源错误；不用于建立学习画像。
- Schema Markup Validator 和 Google Rich Results Test：验证 JSON-LD。
- PageSpeed Insights / Lighthouse：上线前与发布后性能抽检。

### 10.1 核心指标

- 有效索引页面数与应索引页面数一致。
- 重点中心页和视频页无抓取、canonical 或结构化数据错误。
- 品牌查询与非品牌主题查询的展示和点击趋势。
- 视频页的展示、关键时刻识别和点击情况。
- 初学入门、套路教学、古谱研究、人物资料四个集群的覆盖情况。
- 每季度人工检查一组固定问题，记录主要生成式搜索是否引用本站、引用哪一页以及是否准确；不把单次回答当成稳定排名。
