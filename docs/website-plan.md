# 双手剑公开科普网站设计方案

更新日期：2026-07-20

## 1. 产品定义

一句话定位：以于承惠双手剑为主线，把原始教学影像拆成初学者可以观看、理解和跟练的公开课程，并提供有出处的古谱背景。

正式名称：**于承惠双手剑**。

它不是武术资讯门户，也不是宣传型落地页。用户打开网站后应立即进入学习内容。

### 1.1 核心目标

- 降低旧教学视频的学习门槛。
- 保存并清楚传播于承惠双手剑的动作、歌诀与训练思想。
- 让初学者分清基本技法、单个动作、动作组、套路段落和整套演练。
- 用原典、现代考释和编辑示意的来源标签建立可信度。
- 让《武备志·朝鲜势法》成为可阅读、可比较的第二层专题。

### 1.2 明确不做

- 不注册、不登录。
- 不记录或保存学习进度。
- 不使用 `localStorage`、学习 Cookie 或用户画像保存课程状态。
- 首版不开放评论、投稿、社区和用户上传。
- 不把不同传承、古谱复原或现代套路混写成唯一标准答案。

## 2. 内容架构

网站采用五个一级入口，手机与桌面保持同一层级：

1. **开始学习**：课程地图和推荐起点。
2. **基本技法**：安全、器械、握法、身步及击刺格洗。
3. **四段套路**：四段 40 动作、分组教学和整套跟练。
4. **古谱专题**：《武备志·朝鲜势法》二十四势及源流说明。
5. **人物与资料**：于承惠生平、歌诀、参考书目、版权和勘误。

### 2.1 推荐学习顺序

```text
练习前准备
  -> 器械与握法
  -> 击 / 刺 / 格 / 洗
  -> 第一段
  -> 第二段
  -> 第三段
  -> 第四段
  -> 整套跟练
  -> 行功歌诀与剑论
```

网站不保存完成状态。课程地图始终展示完整结构，学习者自行选择入口。

## 3. 页面设计

### 3.1 首页 / 开始学习

首屏直接展示实际学习体验：

- 网站名称和一句简短定位。
- 授权视频中的于承惠持剑全身画面，人物与剑完整可辨。
- 主要操作“从基础开始”。
- 四段套路入口和“观看整套演示”次要操作。
- 下方紧接课程地图，让首屏底部露出下一段内容。

首页后续内容依次为：

- 击、刺、格、洗四法入口。
- 四段套路概览。
- 《双手剑》单练行功歌诀和《双手剑特点与演练要诀歌（剑论）》入口。
- 于承惠人物简介与时间线入口。
- 《武备志·朝鲜势法》专题入口。
- 资料来源和内容方法说明。

首页不展示虚假的学习人数、好评、排行榜或进度百分比。

### 3.2 基本技法总览

- 练习安全和场地要求。
- 剑的结构、长度与训练器械说明。
- 双手握法、基本站姿、身法和步法。
- 击、刺、格、洗四项技法。
- 每项技法使用“定义 + 10-20 秒视频循环 + 三帧动作图 + 常见问题”的固定结构。

四法使用不同的图形标识和辅助色，但必须同时显示文字，不能只靠颜色区分。

### 3.3 四段套路总览

- 顶部展示四段的完整路线和动作组数量。
- 每段为一个独立章节，展示动作组、预计观看时长和对应视频区间。
- 40 个动作使用连续编号，动作组与段落编号并存。
- 页面提供“整套演示”“分段教学”“行功歌诀”三个视图标签。

由于动作名仍需人工校勘，设计阶段只确定结构，不提前固化未经复核的名称。

首版动作名称按视频字幕录入，所有尚未经过人工复核的段落、动作组和动作均显示“待校订”文字标签。该状态不能只用颜色表达。

### 3.4 单课学习页

这是全站最重要的页面。

桌面布局：左侧为视频和控制区，右侧为当前动作组目录；下方为动作图解和文字说明。

手机布局：视频在页面顶部，动作目录改为可展开章节列表；视频不长期悬浮遮挡内容，提供“返回示范”按钮快速回到播放器。

播放器需要：

- 0.5x、0.75x、1x 倍速。
- 镜像模式，并明显提示左右已反转。
- 当前动作或动作组的循环播放。
- A/B 区间循环。
- 前一动作、后一动作。
- 精确到视频帧的前进/后退控制，按约 20fps 处理。
- 中文字幕和可展开文字稿。
- 全屏跟练模式。

播放器下方内容：

- 动作名称、所属段落与序号。
- 起势、过渡、定势三帧图。
- 剑尖轨迹、步法和身体转向图。
- 动作要领与常见问题。
- 该动作使用的基本剑法标签。
- 视频来源和编辑说明。

页面不提供“已完成”按钮，也不记录访问历史。

### 3.5 双手剑歌诀页

歌诀页是四段套路下的独立内容入口，网址可直接分享。页面完整呈现视频最后的两段歌诀：

1. 《双手剑》单练行功歌诀，约对应视频 17:20-18:50。
2. 《双手剑特点与演练要诀歌（剑论）》，约对应视频 18:50-19:55。

页面设计：

- 顶部显示歌诀名称、作者署名、视频区间、来源和“待校订”状态。
- 桌面端采用左侧视频、右侧逐句正文；手机端为视频在上、正文在下。
- 每句歌诀前显示时间码，点击后跳转到视频相应位置并播放。
- 播放时当前句使用左侧实线和字重突出，不依赖底色或动画。
- 提供“连续播放本篇”和“循环当前句”两个明确操作。
- 原字幕转录、后续校订稿和校订说明分层展示，避免静默改字。
- 疑字使用 `〔待辨〕`，页面顶部解释该标记含义。
- 歌诀正文采用易读横排；古籍式竖排仅可作为可选展示，不作为默认阅读方式。

歌诀正文数据存于 `src/content/verses/`。

### 3.6 于承惠人物页

- 以可核验年份构成纵向时间线。
- 重点介绍习武经历、双手剑创传过程、影像传播和相关出版物。
- 采用授权影像截图、出版物封面和资料照片，不生成仿真的历史照片。
- 每个生平节点都有出处，不能用网络称号替代事实说明。

### 3.7 《武备志·朝鲜势法》专题页

专题首页先解释：古谱二十四势与于承惠四段四十动作是两个内容系统。

页面结构：

- 《武备志》原文导读。
- 击、刺、格、洗术语说明。
- 二十四势索引，可按技法筛选。
- 每一势的原图、原文、现代标点、白话说明和编辑备注。
- 古谱与现代套路的“可比较点”，只讨论有依据的术语和技术思想。
- 版本、争议和待考问题。

每势的来源状态必须可见：古籍原文、现代考释或编辑复原。

### 3.8 资料与勘误页

- 参考书目、古籍版本、视频来源和图片授权。
- 本站内容分级方法。
- 已知争议和未决问题。

## 4. 插图与视觉资料体系

插图以“帮助练习者看懂动作”为判断标准，不做纯装饰性古风背景。

### 4.1 首版插图清单

| 类型 | 数量建议 | 制作方式 | 用途 |
| --- | ---: | --- | --- |
| 课程主视觉 | 1 | 授权视频关键帧修复、裁切和调色 | 首页第一视觉 |
| 剑体结构图 | 1 | 实物照片或编辑绘图加标注 | 器械基础 |
| 双手握法图 | 2-3 | 视频关键帧配局部放大 | 握法说明 |
| 四法图解 | 4 组 | 起止帧、剑路箭头、方向标记 | 击刺格洗 |
| 40 动作图 | 每动作 2-3 帧 | 从授权视频抽帧并统一处理 | 套路教学 |
| 四段步法图 | 4 | 足印、方向箭头、转身标记 | 空间路线 |
| 古谱图版 | 24 | 已归档知乎参考图；上线前追查并替换为已核权公版底本 | 朝鲜势法 |
| 人物时间线图 | 按资料确定 | 授权照片、出版物和影像截图 | 于承惠生平 |

### 4.2 教学图形语言

- 红色实线箭头：剑尖或剑身运动方向。
- 蓝色双线弧：腰、肩或身体旋转。
- 绿色足印与编号：落脚位置和先后顺序。
- 虚线：回收路线或画面外延伸方向。
- 圆点：轨迹起点；箭头：轨迹终点。
- “左 / 右 / 前 / 后”文字始终保留，不能只依赖颜色。

镜像模式中的示意图需要同步翻转，并显示“镜像视角”标签。

### 4.3 真实性规则

- 历史人物与史实页面不使用 AI 生成图冒充照片。
- 视频抽帧标记为“原始影像”。
- 从视频描摹或简化的动作图标记为“编辑示意”。
- 古籍影印保留版本和页码信息。
- AI 可辅助清理背景、统一线稿和生成非史实的教学辅助元素，但需要人工复核动作结构。

## 5. 视觉设计系统

整体气质为“训练场的清晰度 + 古籍研究的克制感”，避免影视武侠海报、金色奢华、满屏水墨和营销式大标题。

### 5.1 色彩

建议使用多色但克制的功能色：

| 角色 | 建议色 | 用途 |
| --- | --- | --- |
| 墨黑 | `#171918` | 主文字、深色区域 |
| 纸白 | `#F7F7F4` | 主背景，不使用泛黄仿古底纹 |
| 朱红 | `#A6382C` | 主要操作、剑路 |
| 松绿 | `#2F6255` | 步法、成功状态 |
| 钢蓝 | `#3F6678` | 身体旋转、资料链接 |
| 石灰 | `#D8DAD5` | 分隔线与次级表面 |

所有文字与背景满足 WCAG AA 对比度。红绿不作为唯一的信息差异。

### 5.2 字体

- 界面与正文：`Noto Sans SC` 或系统中文黑体。
- 古籍标题与引文：`Noto Serif SC`，仅在内容层使用。
- 正文手机端不小于 16px，行高 1.6 左右。
- 时间码和动作编号使用等宽数字，防止播放器布局抖动。
- 不使用负字距，不随视口连续缩放字号。

### 5.3 版式与组件

- 采用 4/8px 间距体系。
- 卡片圆角不超过 8px，只用于课程项、资料项和工具区域。
- 页面章节使用无框布局，不把整页堆成卡片墙。
- 桌面正文最大宽度约 1200px，长文阅读列控制在 65-75 个字符。
- 图标统一使用 Lucide 线性图标，并为不熟悉的图标提供悬浮说明。
- 所有点击区域至少 44 x 44px，按钮之间至少留 8px。
- 只使用 150-300ms 的状态过渡，并尊重 `prefers-reduced-motion`。

## 6. 响应式与导航

### 6.1 手机

- 顶部显示网站名和菜单按钮。
- 一级导航使用四项底部导航：学习、套路、古谱、资料；人物内容归入资料。
- 播放器保持稳定宽高比，页面不得横向滚动。
- 横屏时可进入专注跟练布局，控制按钮避开系统安全区。

### 6.2 平板与桌面

- 顶部横向导航展示五个一级入口。
- 课程页使用视频与章节目录双栏布局。
- 古谱页面可使用左侧目录和右侧正文，但正文只保留一个主滚动区域。
- 三层以上内容提供面包屑和稳定返回路径。

所有课程、动作和古谱条目拥有独立 URL，便于公开传播和直接引用。

## 7. 内容数据模型

建议将内容存为结构化数据和 Markdown/MDX，而不是写死在组件中。

核心实体：

- `Person`：人物与时间线。
- `Source`：书籍、古籍、影像和网页来源。
- `Technique`：击、刺、格、洗等基本技法。
- `RoutineSection`：套路段落。
- `ActionGroup`：视频中的动作组。
- `RoutineAction`：40 个具体动作。
- `VideoSegment`：起止时间、字幕和循环范围。
- `VerseCollection`：歌诀标题、署名、来源和校订状态。
- `VerseLine`：歌诀单句、起止时间码、疑字和校订备注。
- `ManualForm`：古谱二十四势。
- `Illustration`：图片、标注、来源和替代文本。

每条内容至少包含：标题、简述、正文、来源、来源层级、审核状态、相关媒体和更新时间。

## 8. 推荐技术方案

项目当前没有现成前端工程，推荐采用静态优先架构：

- `Astro + TypeScript`：生成公开内容页，提供良好 SEO 和较少客户端脚本。
- `React island`：只用于视频播放器、A/B 循环、镜像和古谱筛选等交互区域。
- `Astro Content Collections / MDX`：管理课程、人物和古谱内容。
- `CSS variables`：实现语义化设计令牌。
- `Lucide`：统一界面图标。
- 原生 `HTMLMediaElement` 配合小型播放器模块；如使用 HLS，再引入 `hls.js`。

网站不需要数据库和认证服务，也不写入浏览器持久化存储。

### 8.1 视频发布

原视频约 242MiB，不应直接作为唯一线上版本：

- 保留原始文件作为母版，不在构建时重复复制。
- 生成网页优化的 H.264 MP4，启用 `faststart`。
- 根据实际码率生成一个清晰版本和一个低带宽版本，不对 848 x 464 源片无意义放大。
- 视频放在对象存储/CDN，页面代码与媒体分开部署。
- 生成 WebVTT 字幕、章节文件和海报帧。
- 所有视频区域预留宽高比，避免加载时页面跳动。

### 8.2 公开传播与检索

- 为人物、课程、动作和古谱生成静态详情页。
- 输出 sitemap、Open Graph 图片和规范链接。
- 使用 `Course`、`VideoObject`、`Clip`、`Person`、`Article` 等结构化数据，并保证与可见正文一致。
- 为四法、四段、动作组、整套演示和两篇歌诀标记视频关键时刻。
- 重要事实使用可见来源、校订状态和稳定锚点，便于传统搜索和生成式搜索准确引用。
- 标题和摘要采用事实性描述，避免“最后剑圣”等无法严格证明的宣传话术。
- 默认不接入第三方广告和行为分析；如后续增加统计，应采用匿名、无 Cookie 方案并公开说明。

完整页面矩阵、GEO 内容规则、抓取政策和上线验收见 `docs/seo-geo-plan.md`。

## 9. 无障碍和安全

- 全站支持键盘操作、可见焦点和跳转到正文。
- 视频必须有字幕和文字稿，动作图必须有替代文本。
- 播放器不能只依靠悬浮状态，手机触控也能完成全部操作。
- 文字对比度至少 4.5:1，图形信息对比度至少 3:1。
- 页面支持系统字体放大和减少动态效果。
- 安全页说明练习空间、热身、钝器训练剑、防护距离及未成年人监护。
- 招式说明以文化传承和规范练习为目的，不渲染伤害效果。

## 10. 分阶段实施

### 阶段一：内容与原型

- 人工校勘视频时间轴和 40 动作名称。
- 完成信息架构、首页、课程页和单课页低保真原型。
- 制作一组四法示范图和一个完整动作组，验证插图语言。
- 校订两篇歌诀底稿，并确定标志字和公开署名方式。

### 阶段二：可公开使用的首版

- 首页、基本技法、四段套路、单课学习页。
- 于承惠人物页。
- 朝鲜势法专题首页及二十四势索引。
- 双手剑歌诀页及逐句视频跳转。
- 视频倍速、镜像、章节跳转和区间循环。
- 字幕、文字稿、来源标签和参考资料页。
- 响应式、无障碍、SEO 和媒体优化。
- robots、内容/视频 sitemap、canonical、页面元数据和 SEO/GEO 验收。

### 阶段三：内容深化

- 完成 40 动作全套教学图。
- 补齐二十四势高清古谱图、现代标点和考释。
- 增加古谱与现代套路的术语对照，不做未经证实的一一映射。
- 增加站内搜索、打印版讲义和勘误流程。

## 11. 开工前仍需确定

以下事项不会阻塞内容整理，但应在视觉原型前确定：

1. 是否有可使用的于承惠人物照片、书籍封面或原始光盘封面。
2. 是否有熟悉这套套路的人负责最终动作校对。
3. 公开署名、视频来源和再编辑授权的具体写法。
4. 部署域名和视频对象存储方案。
