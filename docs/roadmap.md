# 于承惠双手剑路线图

更新日期：2026-07-20

状态：开发前基线

## 1. 路线图原则

- 初学者学习主线优先于研究资料深度。
- 先验证页面与插图语言，再批量制作四十动作内容。
- 允许“待校订”内容公开，但状态、来源和疑字必须可见。
- 未核权的古谱图片不得进入生产站点。
- 不以登录、学习进度或用户画像作为产品能力。
- 每个里程碑必须满足退出条件后才能进入下一阶段。

## 2. 里程碑总览

| 里程碑 | 目标 | 主要产出 | 退出条件 |
| --- | --- | --- | --- |
| M0 开发准备 | 建立可执行基线 | 需求、技术设计、路线图、任务卡、内容规范、验收表 | 文档审批通过，风险与占位项已记录 |
| M1 交互原型 | 验证学习流程与视觉语言 | 首页、单课、歌诀、古谱专题四类响应式原型 | 375/768/1440px 无结构问题，核心学习路径可走通 |
| M2 内容与媒体骨架 | 建立可维护的数据和媒体管线 | Content Collections、来源模型、视频清单、字幕、海报、样例插图 | 构建校验能阻止缺号、越界时间码和无许可图片 |
| M3 可用首版 | 完成公开网站主要功能 | 入门、四法、四段、动作页、歌诀、人物、古谱索引、资料页 | 功能、移动端、无障碍和构建测试通过 |

## 3. M0 开发准备

### 范围

- 固化初学者优先、研究资料第二层的信息架构。
- 记录“不登录、不保存学习进度”的产品边界。
- 定义内容状态、来源等级、图片权利和发布门。
- 确定 Astro 静态优先与 React 播放器岛的技术方案。
- 建立任务依赖和上线验收范围。

### 退出条件

- `requirements.md` 与 `design.md` 已审批。
- `tasks.json`、本路线图、内容规范和发布清单已审批。
- 正式署名、域名、对象存储、专业校订人列为上线前依赖，不阻塞本地开发。

## 4. M1 交互原型

### 页面样本

1. 首页：首屏学习入口、课程地图和四法入口。
2. 单课页：视频、章节、倍速、镜像、循环和动作图解区域。
3. 歌诀页：逐句时间码、待辨标记和视频联动。
4. 朝鲜势法专题页：二十四势索引、来源边界和无图版降级。

### 插图样本

- 一项基本技法：验证起止帧和剑路标注。
- 一个完整动作组：验证连续关键帧、步法和镜像。
- 一势古谱：验证原文、现代标点、释义和图版权状态。

### 退出条件

- 页面在 375px、768px、1024px 和 1440px 可读、可操作。
- 首屏不做营销页，直接呈现学习任务且露出后续课程。
- 所有触控目标至少 44 x 44px。
- 信息不依赖颜色，待校订和镜像状态均有文字。
- 样例插图获得内容与动作方向上的人工认可后才允许批量生产。

## 5. M2 内容与媒体骨架

### 内容产出

- 来源、技法、段落、动作组、动作、歌诀、人物和古谱 Schema。
- 四段与四十动作占位目录，字幕名称标记为待校订。
- 两篇歌诀逐句数据和疑字记录。
- 二十四势名称与顺序数据；图版仍保持研究参考状态。

### 媒体产出

- 网页优化 MP4 和低带宽版本。
- WebVTT 字幕、章节清单和播放器片段数据。
- 首页、四法、四段、整套演示及歌诀海报帧。
- 三类教学插图样本。

### 退出条件

- 构建能校验 4 段、40 动作和 24 势无缺号、无重复。
- 视频片段满足起点小于终点且不越过媒体时长。
- `research-only` 和 `unknown` 图片无法进入公开构建。
- 母版视频不会被复制到站点发布目录。

## 6. M3 可用首版

### 页面范围

- 首页与开始学习。
- 入门安全、器械、握法、身步。
- 击、刺、格、洗总览及单项课程。
- 四段总览、段落页、动作页和整套跟练。
- 两篇歌诀独立页面。
- 于承惠人物页。
- 朝鲜势法专题、二十四势索引和可用的单势文字页。
- 来源、版本、许可、校订与勘误页。

### 功能范围

- 0.5x、0.75x、1x 倍速。
- 镜像及持续状态提示。
- 动作区间循环、A/B 循环、前后动作和帧进退。
- 字幕、文字稿、时间码跳转和视频故障降级。
- 移动端四项底部导航和桌面五项顶部导航。

### 退出条件

- 关键用户流程端到端测试通过。
- 页面无登录、进度、收藏或持久化学习状态。
- 所有公开内容显示来源、校订状态和更新时间。
- 键盘导航、可见焦点、字幕、alt 和减少动态效果通过检查。

## 7. M4 发布候选

### 发布准备

- 配置正式域名、canonical 和媒体 CDN。
- 确认整理者署名、视频来源及授权说明。
- 输出 robots、内容 sitemap 和视频 sitemap。
- 输出与可见正文一致的 JSON-LD。
- 完成 Open Graph 图片和错误页。
- 接入搜索平台验证；聚合统计保持无 Cookie、无用户画像。

### 阻塞性退出条件

- 正式域名和 HTTPS 可用。
- 生产视频支持 Range 请求，海报和字幕 URL 稳定。
- 所有生产图片权利状态为 `cleared` 或 `public-domain`。
- 不再显示“来源与授权说明待补充”的占位文字，除非页面明确不进入公开发布。
- 结构化数据、sitemap、canonical、404、响应式和无障碍检查通过。
- Lighthouse 目标：LCP <= 2.5s、INP <= 200ms、CLS <= 0.1；如有偏差必须有记录和修复计划。

## 8. M5 内容深化

- 分批完成四十动作的起势、过渡、定势和轨迹插图。
- 寻找并替换为可核验公版《武备志》图页。
- 增加二十四势现代标点、释义和版本比较。
- 在有证据时增加术语与技术思想比较，不建立未经证实的一一对应。
- 增加站内搜索、打印讲义和公开勘误流程。
- 建立季度内容校订和来源复查。

## 9. 外部依赖

| 依赖 | 最晚需要时间 | 当前处理 |
| --- | --- | --- |
| 正式域名 | M4 | 使用环境变量占位，不写死临时域名 |
| 视频对象存储/CDN | M4 | 本地使用低码率样片开发 |
| 正式整理者与授权文案 | M4 | 暂用“本站整理 / 来源与授权说明待补充” |
| 动作与术语校订人员 | M5 批量插图前 | 页面保留“待校订”状态 |
| 朝鲜势法公版底本 | M4 图片发布前 | 仅发布文字索引或图片占位 |

## 10. 变更控制

- 需求范围变化先更新 `requirements.md`，再评估设计和任务影响。
- 技术方案变化先更新 `design.md`，再调整未执行任务。
- 已发布文字发生校订时保留修订说明、依据和日期。
- 路由发布后原则上保持稳定；必须修改时提供 301 和 canonical 更新。
